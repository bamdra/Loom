# 🧭 把意图说清楚的几种方式

"意图"就是**你跟它说"我想做什么"的那句话**。

它不是命题作文，也不是技术规格。**它是你想要的样子**，用你自己的话说出来就行。下面三个例子，从轻到重，告诉你不同情况下大概要说到什么程度。

> 💡 **想不清楚是常态**。如果你只有一个模糊的想法，别硬挤——直接告诉它"我大概想做 xxx，但 yyy 我没想好"，它会反问你。**它问的问题，就是你需要回答的全部**。

## 🪶 轻的：一句话就够

> 我想给我那个读书摘录页面，在右上角加一个"今天看了什么"的按钮。

这种事：
- 边界很清楚（就一个按钮）
- 没有需要权衡的选择
- 你只想要它做完

写到这个程度就够了。**多写约束反而会让它在你不在意的地方折腾。**

## 🌿 中的：把背景交代一下

> 我那个读书摘录页，现在能按主题归类。我想加一个"按月份看"的视图——
> - 数据已经存了每条摘录的时间
> - 不要破坏现在按主题归类的功能，可以共存
> - 月份视图想要那种像日历一样的格子样式

这种事：
- 涉及好几个地方
- 你有一些**它猜不到**的偏好（"像日历那种格子"）
- 你有不能动的东西（"不要破坏现在的主题分类"）

写得长一点，但每一条都是它**看不到你脑子里就推不出来的事**。约束是告诉它"哪里别动"，不是替它做决定。

## 🌳 重的：把开放问题摆出来

> 我想把这个个人摘录工具，扩成一个能让朋友也能登录、也能看到我推荐内容的小社交场。
>
> 背景：
> - 现在所有数据都在我自己的浏览器里，没有"用户"的概念
> - 我希望朋友登录后能看到我标记为"公开"的摘录，也能给我留评论
> - 不要太复杂——不需要点赞、关注那些
>
> 我还没想好的事：
> - 朋友是用邮箱登录还是用 Google 之类的账号
> - "公开"是按摘录单条选，还是按主题选
> - 我的数据要不要从浏览器搬到一个真正的服务器
>
> 我想先和你聊聊方案，再决定。

这种**比较大的事**，意图反而要"模糊一点"——明确写出"我还没想好的事"，让它知道哪些地方要和你一起讨论。

如果在这个阶段就把所有细节都写死，它只会照搬，等做到一半发现某个选择不合适，回头就麻烦了。

## 🎯 三种情况的共同点

- ✅ **写你已经知道的事**：现有的东西长什么样、不能动什么、你的偏好
- ❌ **不要替它想方案**：除非你真的有强偏好，否则把"怎么做"留给方案讨论
- ❓ **标出你不确定的地方**：你不确定的就明说"我没想好"，让它反问你

意图不是越详细越好，也不是越短越好。**它要匹配你这件事的大小**——小事一句话，大事把背景和不确定的地方摆出来。

## 🔁 它跑偏了怎么办

最快的修正方式不是删掉重来，而是直接告诉它：**"这里走偏了，原因是 X"**。

它会基于已经有的上下文调整，比从零开始更准——它不会把好的部分一起丢掉。

# 🛠️ 高级（可选阅读）

> 📌 **99% 的时间你不需要看这一节，跳过完全没关系。**
>
> 这部分是给"好奇想看一眼底下发生了什么"的人。不影响你正常使用。

如果你打开过左边的文件树，会看到一些目录和文件。它们是 agent 之间沟通用的"会议纪要"和你的项目内容本身。

- **左边的文件树** —— 是项目里的所有文件。点进去能看每个文件的内容。
- **中间的编辑器** —— 你点开一个文件后内容会显示在这里。如果想看 agent 改了什么，可以在这里看；想自己改也可以（不推荐，agent 会失去对你这次改动的追踪，需要你点同步按钮告诉它）。
- **底下的命令行 / 意图框** —— 默认是意图框（你跟 agent 说话的地方）。命令行是给会用的人留的——如果你不会，**直接忽略**就好。

### 你可能会看到的几个特殊文件

它们都在叫 `.vibeflow` 的目录里。打开看看也行，不打开完全不影响：

- 一份**当前在做什么**的清单
- 一份**整体方案**的描述
- 一份**目前进度**的状态

agent 之间靠这几份文件互相传递信息。**它们更新由 agent 自己负责**，你不需要改。

### 撤销·回退

万一某次 agent 做的改动你不满意，所有改动都在你电脑上、你的项目里——可以直接撤销，不会留下烂摊子。如果你不熟悉撤销操作，跟 agent 说："我想把刚才那次改动撤销掉。"它会帮你做。

### 装开发环境

如果它告诉你"我需要 xxx 才能继续"，你不需要自己去搜怎么装——直接说："你帮我装。"它会一步一步告诉你怎么做。
