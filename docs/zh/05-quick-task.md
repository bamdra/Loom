# ⚡ 独立任务 —— 发了就完事

独立任务（Quick Task）用于不需要完整规划流程的小事。小修复、一次性实验、快速调整 —— 输入，发送，搞定。

## 打开独立任务

两种方式：
- 点击 ActionBar 中的**独立任务**按钮（Run 按钮左侧）
- 按 `⌘⇧E`（macOS）/ `Ctrl+Shift+E`（Windows/Linux）

## 写一条任务

弹窗打开后有一个输入框，用大白话写你需要什么：

> 「设置页面的所有按钮加一个深色边框。」

按 `⌘Enter`（macOS）/ `Ctrl+Enter`（Windows/Linux）提交，或点击**保存并执行**。

执行过程：
1. 任务保存到本地文件（`.vibebridge/quick-tasks/tasks.jsonl`）
2. 通过 `/bamdra-vibeflow-run` 发送给运行中的 agent
3. agent 立即执行 —— 不走规划流程

## 任务历史

弹窗底部显示最近的任务（最多 20 条）：
- 每条显示内容、时间、是否已执行
- 勾选「只看未执行」来筛选
- 点击历史条目可重新执行

## 批量执行

要一次跑多条小任务？勾选多条 → 点击**批量执行选中**。

## 独立任务 vs 意图

| 场景 | 独立任务 | 意图（ActionBar） |
|------|---------|------------------|
| 小的独立改动 | ✅ | 大材小用 |
| 需要规划+审核 | ❌ | ✅ |
| 一次性实验 | ✅ | 大材小用 |
| 多步骤功能 | ❌ | ✅ |
| 一句话能说清的 bug | ✅ | 都行 |

**经验法则**：一句话说得清、不动太多文件，用独立任务。否则用意图，让 agent 先规划。
